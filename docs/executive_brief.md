> 受众：公司高层 / 决策层

## 电梯演讲

> **AutoSim 是我们自研的"端到端自动驾驶虚拟测试场"。**
> 它让我们的算法在一座座虚拟中国城市里跑遍各种路况——不用上路、不烧整车——就能验证安全、生成训练数据、做行业里没人做过的对照实验。
> **6 周拿到首份业界标准评测报告，3 个月形成完整对照矩阵。**

---

## 1. 我们要解决什么问题

**端到端自动驾驶（End-to-End AD）**——一个神经网络直接从摄像头看到画面、输出方向盘油门——已是 2025 年行业共识的下一代方案（特斯拉 FSD v12、华为 ADS 4、小鹏 XNGP 都已切到这条路线）。

但这条路线遇到三个根本痛点：

| # | 痛点 | 业界现状 | 我们的策略 |
|---|---|---|---|
| 1 | **训练和验证需要海量"驾驶时间"** | 真车跑一个 corner case 成本高、不可重复、不安全 | 离线仿真，批量跑、可重放 |
| 2 | **现有公开数据 95% 是北美/欧洲场景** | nuScenes、Waymo Open 都不含中国城市 | 接入 ApolloScape 北京 + DrivingDojo + DAIR-V2X |
| 3 | **现有商业仿真器要么闭源、要么不懂中国** | NVIDIA NuRec 闭源、商业街景 ToS 禁用、自研 DSL 6–12 人月 | 在开源 CARLA 基础上自研，可控、可复现 |

**没有一个开箱即用的工具能同时满足"中国路况 + 端到端算法 + 业界标准评分 + 离线批跑"——这就是 AutoSim 要补上的空缺。**

## 3. 这件事的差异化在哪里

业界已有的仿真器很多。AutoSim 的不可替代性来自四个独创点：

### ① 四轨道一致性对照（行业首创）
我们让同一个驾驶算法、同一段道路场景，在四种**完全不同的"画面生成技术"**下分别跑一遍，比较算法表现是否稳定：

```
        同一段道路 + 同一个算法
                    ↓
  ┌────────────┬────────────┬────────────┬────────────┐
  ▼            ▼            ▼            ▼
游戏引擎      下一代游戏    AI 视频生成    真实照片重建
(CARLA UE4)   (CARLA UE5)  (Cosmos)     (3D 高斯泼溅)
```

如果算法在四种"看世界的方式"下表现都一致，说明它真的学会了开车，不是死记硬背画面。**这个对照本身就是一篇可发表的论文**。

### ② 中国路况原生支持
- 用 NVIDIA Cosmos-Predict2.5 + DrivingDojo LoRA 微调，专门生成中国城市街景
- 同时引入 NVIDIA Drive-Dreams（业界最强 AV 视频基座）做对照——这是一个 7B 参数的世界模型，由真车数据后训练得来
- 北京、上海等城市的实景重建（基于 ApolloScape）

### ③ 国际接轨的评分语言
直接对接 **NAVSIM v2 / EPDMS**——CVPR 2025 端到端 AD 挑战赛的官方评分系统。我们的评测结果可以和 Waymo、NVIDIA、Wayve 等公司公开发表的数字直接比较。

### ④ 离线批跑（非实时）
避开了实时仿真的工程陷阱（同步、延迟、确定性），可以一夜跑完几千个 case，做大规模算法消融。

## 4. 架构一图流（业务视角）

```
   ┌──────────────────────────────────────────────────┐
   │  ① 场景定义                                        │
   │     用国际标准格式 OpenSCENARIO 2.0 写"剧本"        │
   │     例：前车突然 cut-in、行人闯红灯、暴雨夜晚       │
   └──────────────────────┬───────────────────────────┘
                          ▼
   ┌──────────────────────────────────────────────────┐
   │  ② 仿真核心（CARLA）                               │
   │     物理引擎 + 交通流 + 自车运动学                  │
   │     每 0.1 秒推进一次世界状态                      │
   └──────────────────────┬───────────────────────────┘
                          ▼
   ┌──────────────────────────────────────────────────┐
   │  ③ 四种"画面生成方式"（同时挂载，可切换）            │
   │                                                  │
   │   游戏引擎 UE4 │ 游戏引擎 UE5 │ AI 视频 │ 真实重建  │
   │     (主基线)   │  (实验)      │ (中国感)│ (黄金对照)│
   │                                                  │
   │   AI 视频内部还分两个 backend，串行跑：             │
   │     • 中国感模型（Cosmos+我们的 LoRA）              │
   │     • AV 高质量基座（NVIDIA Drive-Dreams）          │
   └──────────────────────┬───────────────────────────┘
                          ▼
   ┌──────────────────────────────────────────────────┐
   │  ④ 端到端算法（业界 SOTA，可换）                    │
   │     主：SparseDriveV2 (NAVSIM v2 排行榜 90.38 分)  │
   │     副：Senna（视觉语言模型驾驶版）                 │
   │     输出：未来 3 秒的轨迹                           │
   └──────────────────────┬───────────────────────────┘
                          ▼
   ┌──────────────────────────────────────────────────┐
   │  ⑤ 控制 + 闭环                                    │
   │     轨迹 → 转换为方向盘油门 → 喂回仿真器            │
   │     仿真器更新世界状态，回到第②步                   │
   └──────────────────────┬───────────────────────────┘
                          ▼
   ┌──────────────────────────────────────────────────┐
   │  ⑥ 评测报告                                       │
   │     EPDMS 安全评分 + 四轨道方差 + 多算法横评        │
   │     case × 算法 × 渲染方式 三维矩阵                 │
   └──────────────────────────────────────────────────┘
```

## 5. 核心能力（产品功能清单）

| 能力 | 业务含义 | 第一版交付 |
|---|---|---|

| **真闭环 sim2real** | 用真照片重建的虚拟世界，最贴近真实 | Phase 3 W13+ |

## 8. 风险全景（已识别 + 已缓解）

| 风险 | 概率 | 缓解 |
|---|---|---|
| CARLA 0.10 (UE5) headless 不稳 | 中 | 仅作 A2 副轨道，0.9.16 是 production 主轨 |
| Cosmos 模型 HuggingFace 国内访问慢 | 中 | 镜像源 + Drive-Dreams 双 backend 互备 |
| 国家级数据集（ONCE / DAIR-V2X 全集）审批超期 | 中 | 已有 nuScenes mini + ApolloScape + DrivingDojo 兜底，不阻塞 |
| OpenSCENARIO 2.0 复杂场景能力受限 | 高 | 退化到 Python API（行业通用做法）|
| 四轨道之间渲染差异过大导致算法表现剧烈分化 | 低 | 这本身就是要观察的现象，不是 bug |

**所有红线风险已有 fallback。无单点失败风险。**

## 9. 战略意义

为什么这件事值得公司投入：

1. **团队能力建设**：项目同时涉及自动驾驶、世界模型（Cosmos/Drive-Dreams）、3D 高斯重建（gsplat）三个 2025 年最热的技术方向。做完这一项，团队具备业界顶级 AV 工程能力。

2. **技术储备**：如果公司未来决定进入智能驾驶量产或机器人具身智能，这套仿真链路是直接可复用的核心资产。仿真平台本身就是机器人具身智能（VLA 模型训练）的关键基础设施。

3. **行业影响力**：四轨道一致性对照是业界**首次**——具备发表 CVPR / ICRA / ICLR 顶会的潜力。开源后是公司的技术品牌资产。

4. **不依赖外部**：避免被 NVIDIA NuRec / 商业仿真器供应商绑死。完全自主可控。

5. **低成本试错**：3 个月、2 个人、一台机器，可以拿到一个完整的对照报告。**ROI 角度看，这是公司性价比最高的 AV 技术投入路径。**
